# REST API — Portal CEL Piloto

Single source of truth: `lib/api-spec/openapi.yaml`. Hooks generated by Orval into `lib/api-client-react`. Validation schemas in `lib/api-zod`.

All routes are prefixed `/api`. All require `Authorization: Bearer <Clerk JWT>` unless marked public. Responses are JSON unless noted.

Standard error shape: `{ "error": "code", "message": "human message" }`. Validation errors include `details`.

## Auth tiers

| Tier | Middleware | Who | Used for |
|---|---|---|---|
| Public | none | anyone | `/healthz`, ICS feed by token |
| Auth | `requireAuth` | any signed-in user | Most reads + self mutations |
| PM | `requirePM` | users with `pm_lead` or `pm_cel` role | Editing other members |
| Admin | `requireAdmin` | email allowlist (Camila, Kevin) | Invitations, audit, batch uploads, T0, notifications |

## Endpoint reference

### Health

- `GET /healthz` — public liveness check.

### Self (me)

- `GET /me` — returns `MemberMe { id, email, displayName, roles, isAdmin, ... }`.
- `PATCH /me/display-name` — body `{ displayName }`.
- `GET /me/notifications` — opt-in/out state.
- `PATCH /me/notifications` — body `{ optOut: bool }`.
- `GET /me/roles` — roles assigned to self.
- `GET /me/cv` — current CV metadata.
- `POST /me/cv` — confirm CV upload (body `{ fileName, contentType, sizeBytes, objectPath }`).
- `DELETE /me/cv` — remove CV.

### Team

- `GET /team/members` — list all users with roles.
- `PATCH /team/members/{userId}` — **admin only**. Body `{ displayName?, roles?, clearCv? }`.
- `GET /team/roles` — role catalog.
- `GET /team/summary` — coverage stats: `{ totalRoles, rolesFilled, coverage: [{ role, assignees, tbd, pendingOnCommittee }] }`. Overlays governance.

### Project

- `GET /project/config` — returns `{ startDate }`.
- `PATCH /project/config` — **admin only**. Body `{ startDate: date|null }`. Triggers `ensureSystemWeeklies()`.

### Admin: Invitations

- `GET /admin/invitations` — list.
- `POST /admin/invitations` — body `{ email, suggestedRoles[], expiresAt? }`.
- `DELETE /admin/invitations/{id}` — revoke. 409 if status ≠ `pending`.
- `POST /admin/invitations/{id}/resend` — re-send the email and bump `last_sent_at`.

### Admin: Audit log

- `GET /admin/audit-log?action=&actorEmail=&limit=200` — paginated.

### Admin: Roles

- `GET /admin/roles` — same as `/team/roles` with extra fields for editing.
- `PATCH /admin/roles/{id}` — body `{ description?, sortOrder? }`.

### Admin: Notification recipients

- `GET /admin/notification-recipients`
- `POST /admin/notification-recipients` — body `{ email }`.
- `DELETE /admin/notification-recipients/{email}`
- `POST /admin/notification-recipients/test` — sends a test email.
- `GET /admin/notification-log` — recent log entries.

### Kanban

- `GET /kanban/columns` — list of 5 columns.
- `GET /kanban/cards` — all cards, all categories.
- `POST /kanban/cards` — create. Body `KanbanCardCreate { title, description?, columnKey, phaseId?, assignedRoles?, priority?, dueDate?, category? }`.
- `PATCH /kanban/cards/{id}` — partial update.
- `DELETE /kanban/cards/{id}` — also shifts positions.
- `POST /kanban/cards/{id}/move` — body `{ columnKey, position }`. Server clamps `position` to `[0, columnCount]`.
- `POST /admin/kanban/cards/batch` — **admin only**. JSON `{ cards: [...] }` or multipart CSV. Atomic; per-row errors. Audited.

### Milestones (Calendar)

- `GET /milestones` — all milestones (any kind).
- `POST /milestones` — create one. Blocks `source='system'` edits.
- `PATCH /milestones/{id}` — same.
- `DELETE /milestones/{id}` — same.
- `GET /calendar/export.ics` — public-ish (signed token in URL). Honors `date_override`; if T0 null, exports only `date_override` rows.
- `GET /calendar/feed-url` — returns the signed URL for the user's iCal feed.
- `POST /admin/milestones/regenerate-weeklies` — **admin only**. Re-runs `ensureSystemWeeklies`. Audited.
- `POST /admin/milestones/batch` — **admin only**. JSON `{ sessions: [...] }` or multipart CSV. Atomic; rows go in with `source='import'`. Audited.

### Decisions

- `GET /decisions` — all.
- `POST /decisions` — body `{ title, context, optionsConsidered?, phase?, ownerUserId?, ownerRole?, dueDate? }`.
- `GET /decisions/{id}` — detail.
- `PATCH /decisions/{id}` — partial update.
- `DELETE /decisions/{id}` — hard delete.
- `POST /decisions/{id}/resolve` — body `{ decidedOutcome (required), decidedAt (required), decidedOptionId? }`. Triggers `decision_resolved` email.
- `POST /decisions/{id}/reopen` — clears `resolution / resolved_at / resolved_by` (legacy); preserves `decided_*` fields.
- `POST /admin/decisions/batch` — **admin only**. JSON or multipart CSV. Atomic; audited.

### Documents

- `GET /documents/folders` — fixed folder list.
- `GET /documents?folder=&phaseId=` — list active docs.
- `POST /documents` — confirm upload (metadata).
- `GET /documents/{id}` — detail.
- `PATCH /documents/{id}` — rename / move folder / phase.
- `DELETE /documents/{id}` — soft-delete (`is_active=false`).
- `GET /documents/{id}/download` — signed URL.

### Storage

- `POST /storage/uploads/request-url` — body `{ filename, contentType, kind }` → returns signed PUT URL.

## OpenAPI sections of note

- **`securitySchemes.bearerAuth`**: applied globally; opt out per-route via `security: []`.
- **Multipart**: batch endpoints accept both `application/json` and `multipart/form-data` with a `file` field. CSV is parsed with `csv-parse/sync`; `|` and `;` are accepted as array separators in CSV cells.
- **Date strings**: `date_override` is declared as `type: string, pattern: ^\\d{4}-\\d{2}-\\d{2}$` (not `format: date`) to prevent Zod from coercing to `Date`. Wire format is plain `YYYY-MM-DD`.

## Generated client

import { useListKanbanCards, useCreateKanbanCard } from "@workspace/api-client-react";
const { data: cards } = useListKanbanCards();
